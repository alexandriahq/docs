Once setup is complete, Ambient runs by default.

You do not need to start a session manually. Keep working and Ambient will process the capture sources you enabled into context chunks, intents, memory notes, and handoffs.

## Daily workflow

<Steps>
  <Step title="Work normally">
    Use your editor, browser, terminal, meetings, and docs as usual.

    Ambient observes the sources you enabled and writes local recording outputs.
  </Step>

  <Step title="Check Feed">
    Open **Feed** to see processed context chunks.

    Chunks are the short summaries Ambient creates from recent capture outputs. When a chunk is linked to an intent, Feed groups it under that intent.
  </Step>

  <Step title="Connect private app context">
    Open **Integrations** to connect apps like Todoist, Notion, Gmail, Google Calendar, Granola, and Toggl Track.

    During intent review, Ambient can use read-only integration tools before deciding that information is missing.
  </Step>

  <Step title="Respond to handoff notifications">
    When Ambient creates a handoff, review the ordered steps.

    Accept it to open the task in your configured agent, or deny it if the work should stay with you.
  </Step>
</Steps>

## Watch capture live

Use the live recording overlay when you want visibility into capture itself.

The overlay shows recent finalized recording outputs such as active app/window changes, screen-derived text, audio transcripts, and keyboard activity metadata. It is a capture monitor, not the long-term memory view.

On supported platforms, Ambient uses a native glass overlay. Otherwise it falls back to an HTML overlay window.

<Note>
  Ambient only captures sources you enable and skips the apps, audio sources, and browser URLs you exclude. Update those rules from **Permissions**.
</Note>
