Ambient memory is the running context of your work.

It is built from the capture sources and integrations you enable. Use it to see what Ambient understands before you ask an agent to help.

## Review memory

Open **Memory** from the Ambient app.

Memory shows the work Ambient has learned over time: active goals, recurring projects, useful facts, and recent context. Review it periodically to make sure important work is represented clearly.

## Use the live overlay

Click the pop-out icon to open the live overlay.

The overlay shows recent captures while you work. Use it when you want to confirm that Ambient is running, see what it is learning, or check whether a source is too noisy.

## Keep memory useful

Good memory comes from normal work plus clear capture settings.

<Steps>
  <Step title="Keep working">
    Ambient learns in the background. You do not need to manually start or stop a session.
  </Step>

  <Step title="Check what Ambient learned">
    Open **Memory** after a focused work block or at the end of the day.
  </Step>

  <Step title="Tune capture">
    If Ambient is missing context, enable the relevant source or integration.

    If Ambient is capturing private or noisy work, exclude that app, domain, or workflow.
  </Step>
</Steps>

## When to check memory

- Before asking an agent to continue work.
- After changing capture settings.
- After a long debugging or research session.
- When a handoff notification feels surprising.
