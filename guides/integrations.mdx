Integrations let Ambient pass useful context to the tools where work happens.

Start with your coding agent, then add the work tools you want Ambient to understand.

## Connect a coding agent

Ambient supports connecting agent tools such as Codex and Claude Code.

<Steps>
  <Step title="Open integrations">
    In Ambient, open **Settings** > **Integrations**.
  </Step>

  <Step title="Choose your agent">
    Select **Codex** or **Claude Code**.
  </Step>

  <Step title="Follow the setup flow">
    Ambient will install or configure the local connection it needs.

    Keep Ambient Bridge running while you connect the agent.
  </Step>

  <Step title="Verify the connection">
    Return to **Integrations** and confirm the agent is connected.
  </Step>
</Steps>

## Add more tools

Connect the tools that contain important project context.

Examples include editors, terminals, browsers, issue trackers, docs tools, and communication apps. The more relevant context Ambient can see, the better its memory and handoffs become.

<Tip>
  You do not need to connect everything on day one. Start with the tools you use for active work, then add more as needed.
</Tip>

## Manage an integration

Use **Settings** > **Integrations** to update or remove a connection.

If an integration becomes noisy, disconnect it or adjust capture rules for that app or domain.
