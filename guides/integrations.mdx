Integrations are app connections Ambient can read for private context.

They are not Codex or Claude Code connections. Codex and Claude Code are agent toolchains for handoffs. Integrations are apps such as Todoist, Notion, Gmail, Google Calendar, Granola, and Toggl Track that Ambient can query while maintaining intents.

## Supported apps

Ambient currently exposes first-party integration toolkits for:

- **Todoist** for tasks, projects, labels, and due dates.
- **Notion** for pages, databases, project notes, and workspace search.
- **Gmail** for email threads, senders, unread context, and message search.
- **Google Calendar** for calendars, events, attendees, and meeting times.
- **Granola** for meeting notes, transcripts, decisions, and action items.
- **Toggl Track** for time entries, projects, clients, and workspaces.

## Connect an app

<Steps>
  <Step title="Open integrations">
    In Ambient, open **Integrations** from the sidebar.
  </Step>

  <Step title="Choose an app">
    Select a toolkit such as **Todoist**, **Notion**, **Gmail**, **Google Calendar**, **Granola**, or **Toggl Track**.
  </Step>

  <Step title="Authorize the connection">
    Ambient asks Bridge for a scoped integration token and opens the provider authorization flow.

    Keep Bridge running while the connection completes.
  </Step>

  <Step title="Verify the connection">
    Return to **Integrations** and confirm the app shows an active connection.
  </Step>
</Steps>

## How Ambient uses integrations

During the intent loop, Ambient can use connected integrations before deciding that an intent is blocked by private information.

For example:

- meeting or scheduling questions may need Google Calendar;
- email commitments may need Gmail;
- open tasks or deadlines may need Todoist;
- specs or durable notes may need Notion;
- meeting decisions may need Granola;
- recent project time context may need Toggl Track.

Ambient searches the available read tools, loads the needed schema, and executes read-only calls. Results are used to refine intent summaries, blockers, memory candidates, and handoff readiness.

<Warning>
  Integration reads may contain private account data. Ambient should use only relevant context and should not copy unrelated private content into handoff instructions.
</Warning>

## Manage an integration

Use **Integrations** to reconnect or disconnect an app.

The server controls the supported toolkit allowlist and the read-only tool policy. Write actions are blocked by default.

<Note>
  Toggl Track summaries are managed under **Automations**. When enabled, Ambient can summarize processed context that overlaps Toggl time entries and write that summary into the entry description.
</Note>
