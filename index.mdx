Ambient is a local context layer for your computer.

Install Ambient, choose what it can capture, then keep working. Ambient turns local activity into context chunks, maintains open intents, writes durable memory notes, and prepares handoffs when an agent can continue the work.

## How it works

Ambient separates capture, memory, integrations, and handoffs.

<Steps>
  <Step title="Download Ambient">
    Download Ambient App. Open it, then follow setup to install Bridge through Ambient.

      </Accordion>

      <Accordion title="Windows">
        [Download Ambient App for Windows](https://api.alexandria.so/downloads/apps/ambient-app/alpha/win32/x64/latest.exe)
      </Accordion>
    </AccordionGroup>

    Ambient guides you through installing Bridge during setup. Bridge owns sign-in and network egress; Ambient App owns local capture, storage, processing, memory, and handoffs.
  </Step>

  <Step title="Choose what to capture">
    Enable local sources such as screen, microphone, system audio, keyboard metadata, active app/window, and accessibility context.

    Exclude private apps, media apps, and browser URLs before capture starts.
  </Step>

  <Step title="Build context">
    Ambient writes redacted recording outputs, groups them into context chunks, and tracks open intents over time.
  </Step>

  <Step title="Hand off work">
    When an intent is specific enough for an agent, Ambient creates a reviewed handoff with ordered steps.
  </Step>
</Steps>
